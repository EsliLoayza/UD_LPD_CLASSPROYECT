Semana 02: ¡Dando Inteligencia a tus Aplicaciones!

¡Hola de nuevo, creador de software!

La semana pasada aprendimos a recibir información. Hoy, vamos a darles un cerebro a nuestras aplicaciones. Aprenderás a programar la capacidad de tomar decisiones, haciendo que tus programas sean mucho más interactivos y útiles.

1. Marco Teórico: El Arte de Decidir

a. Ofreciendo Opciones: RadioButton y CheckBox

Para que el usuario elija de opciones predefinidas, usamos:

  • RadioButton: Para seleccionar una sola opción de un grupo (como en un examen de opción múltiple). Se agrupan con un `GroupBox`.
  • CheckBox: Para seleccionar cero, una o varias opciones (como los ingredientes de una pizza).

Ambos controles tienen una propiedad clave: .Checked, que es true si está seleccionado y false si no lo está.

b. El Cerebro del Programa: La Estructura if-else

Permite que tu código ejecute diferentes bloques de instrucciones según una condición. Es la base de la lógica en programación. 

// La estructura if-else if-else evalúa múltiples condiciones en orden.
if (nota >= 18)
{
    // Código si la nota es excelente.
}
else if (nota >= 13)
{
    // Código si la nota es aprobatoria pero no excelente.
}
else
{
    // Código si la nota es desaprobatoria.
}

2. Actividad de Aprendizaje N° 02

Ejercicio: Configurador de Pedidos de Pizza

Crea una aplicación que permita al usuario elegir el tamaño de una pizza (usando `RadioButtons`) y agregar ingredientes extra (usando `CheckBoxes`). Un botón debe calcular y mostrar el precio total del pedido basándose en las selecciones del usuario.

Pista: Necesitarás una variable `double` para ir acumulando el precio y varias sentencias `if` para tomar las decisiones.
