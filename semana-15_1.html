Semana 15: Proyectos Individuales

Unidad: Lenguaje de Programación Distribuida (C# y MySQL)

A continuación, se presentan 15 proyectos individuales diseñados para evaluar de forma integral las competencias adquiridas. Proyecto 01

Sistema de Gestión de Viveros y Trazabilidad Botánica (BotanTrak)

Contexto del Cliente

Un consorcio de viveros especializado en plantas nativas y en peligro de extinción necesita un sistema de escritorio para gestionar su inventario. El problema principal es la trazabilidad: necesitan saber el origen exacto de cada semilla (lote, colector, fecha, geolocalización de recolección), su proceso de germinación, trasplantes, y su ubicación final (venta o reforestación).

Requerimientos Clave

  • Módulo de "Lotes de Semillas" con trazabilidad de origen (colector, coordenadas GPS, fecha).
  • Módulo de "Germinación" para registrar tasas de éxito, fechas y condiciones (sustrato, invernadero).
  • Módulo de "Inventario de Plantas" que vincule cada planta a su lote de semilla original.
  • Gestión de "Invernaderos" y "Secciones" para ubicación física de las plantas.
  • Módulo de "Salidas" (Venta, Donación, Reforestación) que cierre el ciclo de trazabilidad.
  • Reportes de inventario por especie, tasa de germinación por lote y mapa de origen de semillas (simulado).

Entregables Técnicos ObligatorIOS

  • Diseño: Diagrama de Clases completo y Diagrama de Componentes.
  • Arquitectura: Implementación estricta de 3 capas (Presentación: WinForms, Lógica: C#, Datos: C# con MySQL).
  • DLLs: Generación de las bibliotecas de vínculos dinámicos (.dll) para la capa de Lógica y la capa de Acceso a Datos.
  • Base de Datos: Script SQL de creación de la base de datos MySQL (tablas, relaciones, vistas si es necesario).
  • Ejecutable: Entrega del archivo .exe funcional y el código fuente completo.

Sistema de Gestión de No Conformidades y Acciones Correctivas (Q-CapaSys)

Contexto del Cliente

Una empresa de manufactura mediana (ej. textiles técnicos) necesita formalizar su proceso de Calidad (QA/QC). Actualmente, los "reportes de no conformidad" (producto defectuoso, error de proceso) se manejan por email y Excel, perdiendo el seguimiento. Se requiere un sistema de escritorio para gestionar el ciclo de vida completo de una No Conformidad (NC) y su Acción Correctiva/Preventiva (CAPA).

Requerimientos Clave

  • Módulo de "Usuarios" con roles (Operario, Supervisor de Calidad, Gerente).
  • Formulario de "Registro de NC" (fecha, área, descripción, tipo de defecto, lote afectado, evidencia - path a imagen).
  • Gestión de "Workflow de Estado" de la NC (Abierta, En Análisis, Cerrada, Verificada).
  • Módulo de "Análisis de Causa Raíz" (ej. 5 Porqués, Ishikawa - campos de texto).
  • Módulo de "Acciones (CAPA)" vinculado a una NC, con responsable asignado y fecha límite.
  • Dashboard (pantalla principal) que muestre NCs abiertas por área y acciones vencidas.

Proyecto 03

Gestor de Flotillas y Mantenimiento Predictivo (FleetPro)

Contexto del Cliente

Una compañía de logística regional con una flotilla de 30 camiones de carga necesita un sistema para controlar el mantenimiento. Quieren dejar de hacer mantenimientos "correctivos" (cuando algo falla) y pasar a un modelo "predictivo" y "preventivo" basado en el kilometraje (Odómetro) real y el tipo de uso de la unidad.

Requerimientos Clave

  • Módulo de "Unidades" (Camiones) con datos maestros (placa, modelo, año, odómetro inicial).

  • Módulo de "Planes de Mantenimiento" (ej. "Cambio de Aceite cada 10,000 km", "Revisión de Frenos cada 30,000 km").
  • El sistema debe calcular el "km restante" para cada servicio de cada unidad.
  • Generación de "Alertas" (visuales) en un dashboard para unidades que estén a menos de 1,000 km de su próximo servicio.
  • Registro de "Órdenes de Taller" (mantenimientos realizados, costo, repuestos).

Sistema de Administración de Viñedos (VinoTrak)

Contexto del Cliente

Un viñedo boutique necesita un sistema para administrar sus parcelas (lotes de uva). El sistema debe registrar la "variedad" (Malbec, Cabernet) en cada lote, las "labores" (poda, riego, fertilización) realizadas, y finalmente el control de la "vendimia" (cosecha), registrando el peso y nivel de azúcar (grados Brix) de la uva cosechada por lote.
